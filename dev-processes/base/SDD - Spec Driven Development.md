Каким образом лучше всего составлять спецификации для агентов, чтобы они лучше держали контекст.

Общие правила к созданию документации:
- Она должна быть разделена на док. верхнего уровня и документацию нижнего уровня

***Верхний уровень***:
- Описание системы
- Описание клиентов
- Общее описание функционала программы
- Описание наших ресурсов

***Нижний уровень:***
- Описание самих задач
- Может быть описание сейчас / надо
- Каким образом выполнить текущую задачу

## Структура проекта

3 основных файла

```
specs/
|
├── mission.md
├── tech-stack.md
├── roadmap.md
|
├── tasks/
	
```

### mission.md

Описание аудитории, о чем проект, какие проблемы решает, каким функционалом обладает

### tech-stack.md

Технический стек, включая структуру базы данных, различные trade off

### roadmap.md

Верхне-уровневое описание фаз разработки проекта.
